# ⚗️ Automated Lab Safety Auditor

An intelligent, OSHA-compliant chemical formulation auditing system built on a hybrid **RAG + API-based LLMs + SQLite Caching + Data Validation** pipeline. Submit any lab formulation note in natural language and receive a structured compliance report in milliseconds, backed by dynamic web search fallbacks.

## Overview

The Lab Safety Auditor evaluates chemical formulation notes against OSHA regulatory limits and hardware safety thresholds. It produces a structured report with three possible verdicts:

| Status | Meaning |
|--------|---------|
| `✅ APPROVED` | All chemical concentrations and hardware temperatures are within safe limits. No systemic hazards detected. |
| `⚠️ PARTIAL` | Exposure limits are exceeded but represent airborne/inhalation TWA limits (not liquid volume % violations), or a physical hazard exists (e.g., heating solvent above its boiling point). |
| `❌ REJECTED` | A hard OSHA violation (e.g., Benzene > 0.1% by volume) or hardware thermal limit exceeded. |

## New Advanced Features

1. **API-Based LLM Integration & Resilient Fallbacks**
   * Migrated summary generation and narrative parsing from local Ollama to the high-performance **Google Gemini API** (`gemini-3.5-flash`).
   * Configured an automatic fallback connection to **OpenRouter's free router** (`openrouter/free`) utilizing HTTPX to handle transient Gemini 503 or 429 rate limit errors transparently.

2. **Precision Multi-Tier SQLite Caching**
   * **Layer 1 (OSHA Limits Cache)**: Stores parsed OSHA standard limits (ppm/pct/BP) per chemical to eliminate redundant vector searches.
   * **Layer 2 (Exact Query Cache)**: Employs SHA-256 hashing on user inputs. Duplicate formulation requests yield instant sub-millisecond responses without calling RAG, LLM, or Web Search APIs.
   * **Layer 3 (Summary Cache)**: Caches safety summaries generated by the LLM based on violation notes hashes.

3. **Hybrid Narrative Parser**
   * Uses regex for fast, deterministic extraction. If zero chemical entities are found, the pipeline falls back to an LLM narrative parser to extract complex, conversational user input.

4. **Data Validation & Fuzzy Name Correction**
   * Automatically corrects chemical name typos using `rapidfuzz` string similarity.
   * Performs physical sanity checks: flags contradictory units (e.g., `% ppm`) and warns if temperatures exceed hardware failure levels by more than 150%.
   * Displays warnings directly on the UI as styled alert cards.

5. **Dynamic Web Search Fallback (Tavily)**
   * Directly integrates the Tavily Search API. If a chemical has no record in the local RAG database, the pipeline searches the web for OSHA standards, extracts exposure limits using LLM reasoning, and caches the result for future queries.

## Configuration

All local settings live in [`.env`](.env) (or `.env.example`):

| Variable | Default Value | Purpose |
|----------|---------------|---------|
| `LLM_PROVIDER` | `gemini` | Primary LLM provider (`gemini` or `openrouter`) |
| `GEMINI_MODEL` | `gemini-3.5-flash` | Gemini model slug |
| `GEMINI_API_KEY` | `your_key` | Google AI Studio developer API Key |
| `OPENROUTER_API_KEY` | `your_key` | OpenRouter developer API Key |
| `OPENROUTER_MODEL` | `openrouter/free` | Resilient free-tier fallback router model |
| `TAVILY_API_KEY` | `your_key` | Tavily Web Search API key |

## Setup & Running

### 1. Prerequisites
* Python 3.11+
* Active API Keys for Gemini (Google AI Studio) and Tavily.

### 2. Installation
```bash
# Clone the repository and navigate inside
cd L1_Project

# Create and activate virtual environment
python -m venv .venv
.\.venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
```

### 3. Environment Setup
Copy the example environment file:
```bash
cp .env.example .env
```
Open `.env` and fill in your actual developer API keys.

### 4. Database Ingestion (RAG)
Load the baseline OSHA regulatory standards into the ChromaDB vector database:
```bash
python ingest.py
```

### 5. Launch web app
```bash
python run.py
```
Open `http://127.0.0.1:7860` in your web browser.

### 6. Run test suite
The test suite can also be executed via the run script (or directly via tests/test_formulations.py):
```bash
python tests/test_formulations.py
```
*Note: Running this will automatically generate and update `evaluation_results.json`.*

## Test Scenarios

The auditor is evaluated against 22 test scenarios across 8 categories:
1. **Benzene Volume Violations** (exceeding 0.1% volume → REJECTED)
2. **Benzene Exposure** (exceeding 1 ppm TWA limit → PARTIAL)
3. **Acetone Exposure** (exceeding 1000 ppm TWA limit → PARTIAL)
4. **Toluene Exposure** (exceeding 200 ppm TWA limit → PARTIAL)
5. **Methanol Exposure** (exceeding 200 ppm TWA limit → PARTIAL)
6. **Isopropanol Exposure** (exceeding 400 ppm TWA limit → PARTIAL)
7. **Hardware Thermal Safety** (glass/polypropylene/steel maximum limits exceeded → REJECTED)
8. **Web Search Fallback** (real-time OSHA scrapes for dynamic inputs)

## Evaluation Metrics

The Gradio dashboard displays real-time statistics retrieved directly from the last test suite validation run:
* **RAG Context Relevancy**: Accuracy of retrieved standards.
* **Agent Success Rate**: Successful checks performed without crashing.
* **LLM Instruction Following**: Quality and formatting of generated summaries.
* **Total Latency**: Average pipeline processing speed (down to **<0.01 seconds** with active cache hits!).
